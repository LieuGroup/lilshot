# lilshot — Shottr-style editor (fourth milestone, 3 rounds)

After ANY successful GUI capture (picker Enter, ⌥⇧R, ⌥⇧F, ⌥⇧A): keep the existing behavior
(original → clipboard + shutter) AND open the editor window with the captured image. Editing then
Cmd+C replaces the clipboard with the annotated render. CLI stays headless — no editor.

## Architecture

- `EditorModel` (LilshotCore, TDD): tool selection, annotations array, undo/redo stacks
  (snapshot-based is fine), crop state. Pure — CoreGraphics value types only, no AppKit.
- `Annotation` (core): enum — arrow(from,to), rect(rect), text(origin,string,fontSize),
  blur(rect), stepNumber(center,index); each with color + strokeWidth where relevant.
  All coordinates in IMAGE pixel space (not view space) so export is lossless.
- Geometry helpers (core, TDD): arrowhead point math, drag→rect normalization reuse
  (RegionGeometry), crop clamping, view↔image coordinate mapping given zoom-to-fit scale.
- `AnnotationRenderer` (LilshotMac): draws annotations into a CGContext over the base image →
  flattened CGImage for clipboard/save. Same renderer drives the on-screen overlay drawing.
- Editor UI (LilshotApp): NSWindow (titled, resizable), zoom-to-fit canvas, left/top toolbar,
  color swatches (amber #FFB520 default, red, blue, black, white), Esc/Cmd+W close,
  Cmd+C copy flattened (+shutter), Cmd+S save PNG to ~/Desktop with the existing filename pattern,
  Cmd+Z/Cmd+Shift+Z undo/redo. Multiple captures → newest replaces the window content (single
  editor window, keep it simple).

## Round 1 — shell + crop + export

Editor window opens on capture; toolbar skeleton (tool buttons render, only crop active);
crop = drag a rect (dimmed outside, reuse region-overlay drawing idiom), Enter applies, Cmd+Z
reverts; Cmd+C flattened copy; Cmd+S save; window/keyboard lifecycle. EditorModel TDD: tool
state, crop apply/undo, snapshot undo stack.

## Round 2 — draw tools

Arrow (with arrowhead math TDD), rectangle outline, step numbers (filled circle + auto-increment
white numeral, TDD the counter), text (click places an inline NSTextField at image position;
Enter commits to an Annotation; font ~5% of image height). Color palette wired; all tools go
through EditorModel undo. Live drag preview for arrow/rect.

## Round 3 — blur + manipulation + polish

Blur tool (CIPixellate over rect, scale-aware cell size); select tool: click hit-test (TDD) to
select an annotation, drag moves it, Delete removes; tool keyboard shortcuts (V select, A arrow,
R rect, T text, B blur, N number, C crop); zoom 100%/fit toggle (Cmd+0 / Cmd+9).

## Acceptance (each round)

swift test green (TDD list above), release build clean, make-app.sh works, files < ~200 lines,
conventional commits per logical change, no AI refs, dead-code cleanup, no plan references in
code. GUI behaviors listed for manual verification, never claimed.
